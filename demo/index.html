Workflow

Three steps to encrypted delivery

Step 01

Pair your device

Run iris setup on your server. A QR code appears containing the channel ID and relay endpoint. Scan it with the Iris iOS app to exchange public keys. The private key never leaves your iPhone's Secure Enclave.

Step 02

Send encrypted screenshot

Capture a screenshot with your tool of choice, then pipe it through iris push. The CLI generates an ephemeral P-256 key pair, derives a shared secret via ECDH, and encrypts the image with AES-256-GCM. Only ciphertext reaches the relay.

$ iris push screenshot.png
Reading screenshot.png (142 KB)
Generating ephemeral key pair...
Encrypting with P-256 ECIES...
Uploading ciphertext to relay...
Encrypted and pushed. (P-256 ECIES, 142 KB)
TTL: 30 minutes

Step 03

Receive on iPhone

The relay triggers an Apple Push Notification. The Iris app fetches the ciphertext, decrypts it using the Secure Enclave private key, and displays the screenshot. The entire round-trip completes in under a second. After 30 minutes, the relay auto-deletes the ciphertext.

Security Features

Built for zero-trust delivery

P-256 ECIES

E2E Encryption

ECDH key exchange with HKDF-SHA256 derivation and AES-256-GCM authenticated encryption. Each message uses a fresh ephemeral key.

Secure Enclave

Hardware Key Storage

Private keys are generated inside the iPhone's Secure Enclave. They cannot be exported or accessed by the OS, apps, or backups.

QR Pairing

One-Scan Setup

Scan a QR code to exchange public keys and pair your device. No accounts, no passwords, no cloud sync.

APNs

Push Notifications

Real-time delivery via Apple Push Notification service. Screenshots arrive on your lock screen within a second.

TTL: 30min

Auto-Delete

Ciphertext is automatically purged from the relay after 30 minutes. No long-term data retention. Channel metadata expires after 30 days.

Zero Knowledge

Blind Relay

The relay server stores only ciphertext. It holds no decryption keys and has no capability to view image contents.

Key Exchange

ECDH (P-256)

Ephemeral key pair generated per message. Shared secret derived from ECDH with recipient's public key.

Key Derivation

HKDF-SHA256

Derives a 256-bit symmetric key from the ECDH shared secret using HKDF with a context-specific salt.

Encryption

AES-256-GCM

Authenticated encryption providing both confidentiality and integrity. Tamper detection is built in.

Key Storage

Secure Enclave

Private key bound to the device's hardware security module. Cannot be exported or backed up.
